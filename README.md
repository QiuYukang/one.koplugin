# one.koplugin

在 KOReader 上离线阅读「ONE · 一个」（wufazhuce.com）每日更新的**图文 / 文章 / 问答**。

复杂世界，一个就够。

## 安装

把整个 `one.koplugin` 目录放到 KOReader 的 `plugins/` 下，重启 KOReader。
入口挂在 **顶部菜单 → 工具（Tools）标签 → ONE · 一个**（书架与书内均可进入）。

## 功能

- **今日一期**：今日图文/文章/问答 → 生成 EPUB 并打开。当天已生成过则直接复用。
- **最近 7 天**：最近 7 期；可一键合成为一本合集（两级目录）。
- **按日期查看**：昨天 / 上周今天 / 一个月前 / 一年前，或自选日期 / 日期区间。
- **已缓存内容**：离线打开历史生成的期；长按删除单期。
- **设置**：图片质量（600 / 900 / 1080(默认) / 原图）、启动默认打开今日、图片缓存上限、
  启动自动清理与阈值、缓存目录、一键全部清空。
- **书内导航**：可在「手势/按键」里绑定「上一期 / 下一期 / 打开 ONE」动作。

每日图文那张大图会设为 EPUB 封面（书架缩略图），但不占单独一页——正文第一页
直接是图文章节，避免重复。

## 目录结构

```
_meta.lua              插件元信息
main.lua               入口：菜单树 + 业务编排
lib/
  client.lua           HTTP 抓取（v3 JSON GET、HTML GET、二进制下载、限流、重定向）
  v3.lua               v3 JSON API 数据源（映射为统一的图文/文章/问答结构）
  parser.lua           HTML 解析（兜底）+ 富文本 HTML 转有序块
  date_index.lua       image_id ↔ VOL ↔ 日期 换算（锚点插值 + 探测 + 有界扫描）
  image_cache.lua      图片下载与本地缓存（按质量参数缩放）
  epub_builder.lua     手写 zip writer + EPUB 组装（单期 / 合集）
  cleanup.lua          缓存统计、按时间清理、按容量上限清理
  one.lua              服务层：v3 优先、HTML 兜底的抓取 / 缓存 / 组装编排（无 UI）
  settings.lua         设置持久化
  crypto.lua           sha256（图片文件名）
  i18n.lua             中文翻译字典
```

缓存**按日期分文件夹**组织,根目录默认 `<koreader-data>/one/cache/`(可在「设置 → 缓存管理 →
缓存目录」改到 SD 卡等任意可写路径):

```
<cache_dir>/2026-07-07/          每期一个文件夹(按 ISO 日期命名)
  issue.lua                      元数据(清理时保留,可离线重建 EPUB)
  image.jpg / article-1.jpg …    图片(可读文件名)
  ONE VOL.5022.epub              单期 EPUB
<cache_dir>/collections/         合集 EPUB
```

清理策略:自动清理删除超过阈值的日期文件夹里的图片与 EPUB,保留 `issue.lua`;
图片总量超过上限时按最旧优先淘汰。图片质量四档 **600 / 900 / 1080(默认) / 原图**。
「今日一期」若当天已生成过 EPUB 会直接复用,不重复下载。

## 数据来源

**优先使用 v3 官方 JSON API**（`http://v3.wufazhuce.com:8000`，无鉴权），
不可用时**自动回退到 PC 主站 HTML 抓取**（`https://wufazhuce.com`）。

- v3 每条内容自带发布日期字段，所以「按日期 / 按区间」历史查询能拿到**完整三章**
  （图文 + 文章 + 问答）；文章/问答按官方时间线 `reading/index` 定位，图文按日期锚点
  插值 + 探测定位。
- 早期日期（约 2018 年前）问答系统尚未上线，历史期只有图文 + 文章。
- HTML 兜底模式下，文章/问答无日期字段，「按日期」历史期退化为图文单章。

## 合规

抓取本地限流（请求间隔 ≥ 200ms），不上传任何用户数据。图片 URL 统一改写为 https
避免 e-reader 混合内容拦截。本项目仅供个人学习使用，请遵守「ONE · 一个」的用户
协议与相关法律法规。
