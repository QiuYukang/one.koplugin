4. 关键实现要点

UI 组件对应关系（Lua 代码怎么写）

  • 屏 02/03/04/16/17/18/20/22 = Menu:new{ items = {...}, title, subtitle }
  • 屏 05 = DateTimeWidget:new{ ... }
  • 屏 06 = Trapper:info("正在获取 VOL.5022...")
  • 屏 08/14 = crengine 自动生成的 TOC，读取 EPUB 内 nav.xhtml
  • 屏 13/19 = ButtonDialog / MultiConfirmBox
  • 屏 21 = Notification:notify(text, timeout)
  • 屏 12（书内菜单）= 挂载到 ReaderMenu 的 tools 分组 + Dispatcher:registerAction("one_next_issue")

为什么走 EPUB 而不是 HTML/TextViewer

  • 用户想要 KOReader 全套阅读能力：字体/字号/夜间/朗读/词典/高亮/笔记/阅读进度/KOSync
  • 每期一本书 → 自然进书架，KOReader 原生列表 & 搜索直接就用
  • EPUB 内可以有 TOC（NCX + nav.xhtml），图文/文章/问答各占一章
  • 合集 = 多期打包成一本书，同样机制

EPUB 拼装（每期 1 本）

  • 文件结构：mimetype + META-INF/container.xml + OEBPS/content.opf + OEBPS/toc.ncx + OEBPS/nav.xhtml + 3 个 chapter*.xhtml + images/cover.jpg + images/one.jpg（图文大图）+ images/*.jpg（文章问答内嵌图）
  • Lua 打包用 ffi/zlib + 自实现 ZipWriter（mimetype 必须首个且 STORE 无压缩）
  • metadata：dc:title="ONE VOL.5022 · 2026-07-07"、dc:creator="ONE 一个"、dc:date=发布日期 → KOReader 书架按此排序
  • 产物落盘 koreader/settings/one/books/ONE-VOL.5022.epub

EPUB 拼装（N 期合集）

  • N 期 × 3 章 = 3N 章。TOC 两级：一级 VOL/日期，二级 图文/文章/问答
  • book title = "ONE 合集 · 2026-07-01 – 2026-07-07"
  • 封面 = 起始日期图文的封面图

菜单与阅读器的联动

  • ONE 菜单挂在 ReaderMenu / FileManagerMenu 的 tools 分组（书内/书架都能进）
  • 用 ReaderUI:showReader(epub) 打开 EPUB；当前若已有书打开，KOReader 自动关闭切换
  • Dispatcher:registerAction("one_prev_issue" / "one_next_issue")，用户可绑定手势/键位

缓存策略

  • v3 JSON API 的详情响应存 cache/one/<type>_<id>.json（永久）
  • 图片存 cache/one/img/<hash>.jpg（LRU 200MB）
  • 生成的 EPUB 存 settings/one/books/（可手动删）
  • 重开同一期若 EPUB 已存在，直接 ReaderUI 跳过重建
  • 启动自动清理：删 X 天前访问过的 img + EPUB，JSON 保留可离线重建

不做的功能（首版）

  • 评论：网站评论是第三方 JS 挂件，HTML 拿不到
  • 音乐 / 电台 / 电影：ONE 网页版没有，只 App 独占
  • 历史文章 / 问答按日期精确查：文章页缺日期字段，暂只支持"今日跟随首页 3 元组"
